LP側の協賛店導線を修正してください。
今回の目的は、「協賛店の方へ」を押したときに、いきなり登録画面へ飛ぶ誤動線をやめて、正しい説明ページ → 最下部CTA → 協賛店登録画面、という流れに戻すことです。

確認待ちで止めず、
現状確認 → 原因特定 → 修正 → テスト → 最終報告
まで進めてください。

====================================
【問題】
====================================

現在、LPのモーダル内にある
「協賛店の方へ」
を押すと、すぐに協賛店登録画面へ遷移してしまっています。

しかし本来の正しい導線は以下です。

1. モーダルで「協賛店の方へ」を押す
2. LP内の協賛店向け説明ページへ遷移する
3. 内容をしっかり読める
4. 最下部までスクロールする
5. 最下部のCTA
   「協賛店登録ページへ進む」
   を押す
6. そこで初めて本来の協賛店登録画面へ進む

現在この導線が崩れています。

====================================
【今回の目的】
====================================

- 「協賛店の方へ」押下時に、いきなり登録画面へ飛ばないようにする
- まず協賛店向け説明ページを見せる
- そのページの最下部CTAから本来の協賛店登録ページへ進む流れに戻す
- 日本語版 / 英語版の整合を取る
- 既存のLINE相談導線は壊さない

====================================
【正しい導線】
====================================

正しい導線は以下です。

モーダル
「協賛店の方へ」
↓
協賛店向け説明ページ
（内容確認）
↓
最下部CTA
「協賛店登録ページへ進む」
↓
本来の協賛店登録画面

この流れに戻してください。

====================================
【確認対象】
====================================

以下を確認してください。

1. LPモーダル内ボタン
- 「協賛店の方へ」
- 日本語版
- 英語版

2. 関連JS
- ボタン押下時の遷移先
- 共通JS
- button-setup 系
- モーダル制御JS
- audience パラメータ処理

3. 協賛店向け説明ページ
- 正しいURL
- 正しい着地点
- 最下部CTAのリンク先

4. 協賛店登録ページ
- 実際の登録URL
- 日本語版 / 英語版

====================================
【実装要件】
====================================

A. モーダルの「協賛店の方へ」修正
- 直接登録ページへ飛ばさない
- 協賛店向け説明ページへ遷移させる
- 本番LP上で実際に内容が読めるページへ着地させる

B. 協賛店説明ページの最下部CTA確認
- 「協賛店登録ページへ進む」ボタンが正しい登録ページへ進むこと
- CTAが壊れていないこと
- LP → 説明 → 登録の順が成立すること

C. 英語版も整える
- 英語版LPでも同じ考え方にする
- 英語版の協賛店向け説明ページ → 英語版登録ページへ進む流れを成立させる
- 英語版がない場合は、現状の構造に合わせて最も安全な導線にする
