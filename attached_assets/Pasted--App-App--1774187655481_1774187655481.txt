目的：
英語版App登録ページで、送信ボタンを押しても何も起きない不具合があります。
今回は英語版App登録系ページを横断確認し、同様の不具合があればまとめて原因調査と修正をしてください。

最重要：
- 既存の日本語版App登録ページは壊さないこと
- 日本語版の送信処理・API・既存動作に影響を出さないこと
- 今回は見た目変更ではなく、送信処理の不具合修正に集中すること
- 英語版LPではなく、英語版App登録ページ側の不具合調査・修正が目的です

現状確認済みの事実：
- sponsor-registration-en.html の「Submit Registration」ボタンを押しても何も起きない
- Back Home は正常に英語版Appへ戻れる
- 英語版LPから英語版Appへの導線は概ねできている
- 他の英語版登録ページでも同じ症状の可能性がある

調査対象候補：
- app.tomotrip.com/sponsor-registration-en.html
- app.tomotrip.com/guide-registration-v2-en.html
- app.tomotrip.com/tourist-registration-simple-en.html
- それぞれに紐づく英語版JS
- 共通送信処理
- 共通API呼び出し
- 共通バリデーション処理
- フォーム submit イベント
- ボタンクリックイベント

やってほしいこと：

   - sponsor-registration-en.html
   - guide-registration-v2-en.html
   - tourist-registration-simple-en.html

2. それぞれについて以下を確認してください
   - ボタンクリックイベントが接続されているか
   - form submit イベントが接続されているか
   - preventDefault の使い方が誤っていないか
   - バリデーションで止まっている場合、ユーザーにエラー表示が出ているか
   - API送信が発火しているか
   - Console error / Network error が出ていないか
   - 英語版だけ参照ファイル名や id/class 名がズレていないか

   - クリックしてもイベントがないのか
   - submit が発火していないのか
   - 必須項目不足で止まっているのに表示が出ていないのか
   - API送信失敗なのか
   - 英語版だけDOM参照がズレているのか
   - ボタン type 属性が不適切なのか
   - form タグや submit ハンドラの接続先が間違っていないか

5. 日本語版登録ページには影響を出さないでください
   - 日本語版と英語版で分離できるなら英語版のみ修正
   - 共通ファイルを触る場合は日本語版への影響確認を必須にする
